# Tools and Systems

## Daily Tools

**ChatGPT** — Primary thinking partner for planning, ideation, problem-solving, and drafting content.

**Claude** — Used for writing, structured thinking, and refining ideas and messaging.

**Claude Code** — Used for building tools and developing functionality for apps and systems.

**VS Code / Cursor** — Primary development environments for writing and managing code.

**X (Twitter)** — Content platform for posting, engagement, audience building, and distribution.

**WordPress.org** — Website platform for hosting and managing web properties and landing pages.

**MailerLite** — Email platform for writing, managing, and distributing the weekly newsletter.

**Notion** — Organizational system for notes, planning, and structuring information.

**Gmail** — Core communication hub and account management across platforms.

**Microsoft Word** — Writing environment for drafting longer-form content when needed.

**Google Docs** - Alternative writing environment with autosave and easy sharing.

**Google Drive** — Cloud storage for file backup, organization, and remote access.

## Integrations and Connections

**MailerLite ↔ WordPress.org** — Newsletter signup forms are connected to the website to capture subscribers and add them to the email list.

**Manual Workflow (Primary Mode)** — Most tools are currently used independently, with manual movement between systems (e.g., drafting in AI tools, publishing to X or newsletter, managing files in Drive).

**Low Automation by Design (Current State)** — The system is intentionally simple at this stage, prioritizing flexibility, learning, and speed of iteration over automation.

## Data Sources

**Local Documents (Primary Writing Source)** — SWELL newsletters and related documents are created and stored locally on the computer, with backup to Google Drive.

**Google Drive** — Secondary storage and backup system for documents, files, and general assets. Used for remote access and redundancy.

**GitHub** — Code repositories for projects in development. Source of truth for version-controlled code.

**Local Development Environment** — Active code is often worked on locally before being pushed to GitHub or deployed.

**Vercel** — Deployment platform for live web applications (currently hosting at least one active project).

**Notion** — Central hub for notes, planning, and project organization.

**ChatGPT / Claude Projects** — Secondary planning and ideation layer where project thinking, drafts, and structured conversations are stored and iterated on.

## Tried and Rejected

None significant at this time — tool selection has been intentional and relatively streamlined, with no major platforms adopted and later abandoned.
